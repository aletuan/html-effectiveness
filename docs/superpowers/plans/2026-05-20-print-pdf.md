> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bake the canonical `@media print` block into the 10 document-shaped templates (and their `templates/` copies) plus add SKILL.md guidance, so document artifacts "Save as PDF" cleanly.

**Architecture:** The canonical block already lives in `references/design-tokens.md` (committed). For each document template: integrate the block into its `<style>`, append the file's own card/panel class to the `break-inside: avoid` rule, tag screen-only chrome with `.no-print` / `.no-print-sticky`, and reset large body padding under print. Then copy the integrated top-level file over its `templates/` pair so the two are **identical by construction** (avoids the mirror-drift risk). Verify each file under emulated print media.

**Tech Stack:** Plain HTML/CSS, no build system. Verification via a temporary `python3 -m http.server` + Playwright print-media emulation, plus `diff` for mirror parity.

**Spec:** `docs/superpowers/specs/2026-05-20-print-pdf-design.md`

## Shared recipes (used by every per-file task)

### Integration recipe (apply inside the file's `<style>`, at the end)

1. Paste the canonical block **verbatim** from `references/design-tokens.md` → "Print / PDF" section.
2. **Append the file's card/panel selector(s)** to the `break-inside: avoid` rule. Find them by reading the `<style>`: they are the selectors that set a `border` + `border-radius` (the card pattern). Example from the status report: `.stat-card, .chart-panel, .carryover`.
3. **Reset body padding** under print if the file sets a large one: add `body { padding: 0; }` and, if the layout uses a max-width wrapper, `.<wrapper> { max-width: none; margin: 0; }`.
4. **Tag screen-only chrome:** add `class="no-print"` to toolbars/nav/demo controls, or `class="no-print-sticky"` to a pinned element that should still print but flow inline. (Buttons are auto-hidden.)

### Mirror recipe (keep the two copies identical)

```bash
cp <NN-name>.html .claude/skills/html-output/templates/<NN-name>.html
diff <NN-name>.html .claude/skills/html-output/templates/<NN-name>.html   # must print nothing
```

### Verification recipe (print-media emulation)

Start the server once (Task 1, step 1) and reuse it:

```bash
python3 -m http.server 8731 >/dev/null 2>&1 &   # note the PID to kill later
```

Then, via the Playwright MCP, navigate to `http://localhost:8731/<NN-name>.html` and run
(`mcp__playwright__browser_run_code_unsafe`):

```js
async (page) => {
  await page.emulateMedia({ media: 'print' });
  const bodyBg = await page.locator('body').evaluate(el => getComputedStyle(el).backgroundColor);
  const buttonsShown = await page.locator('button').evaluateAll(els =>
    els.filter(el => getComputedStyle(el).display !== 'none').length);
  const noPrintShown = await page.locator('.no-print').evaluateAll(els =>
    els.filter(el => getComputedStyle(el).display !== 'none').length);
  // catches a sticky nav/aside that was never tagged: nothing VISIBLE should stay pinned in print.
  // (must exclude display:none — a .no-print element still computes position:sticky while hidden)
  const stickyPinned = await page.locator('*').evaluateAll(els =>
    els.filter(el => {
      const s = getComputedStyle(el);
      return ['sticky', 'fixed'].includes(s.position) && s.display !== 'none';
    }).length);
  return JSON.stringify({ bodyBg, buttonsShown, noPrintShown, stickyPinned });
}
```

**Expected after integration:** `bodyBg` = `"rgb(255, 255, 255)"`, `buttonsShown` = `0`, `noPrintShown` = `0`, `stickyPinned` = `0`.
Also spot-check in the snapshot that the file's cards carry `break-inside: avoid` (run a one-off
`getComputedStyle(el).breakInside` on a card selector).
