html-effectiveness · project structure

How the repo is laid out

The repo is intentionally flat: 20 numbered .html files at the top level are the actual artifacts, an index.html stitches them into a categorized gallery, and a parallel copy lives inside .claude/skills/html-output/ so Claude Code can use the same files as templates. There is no build step, no node_modules, no shared style.css — each example is a single self-contained page and the duplication is on purpose.

Two halves, one shared spine

1
./*.html — 21 files at the top level

The numbered files are the deliverable. Each one is a complete, double-clickable webpage with its own <style> block inline and no external requests. There is no shared CSS file by design — every page has to stand on its own, so a user who finds 12-incident-report.html through a search engine gets the full thing in one download.

The numbering encodes both the category (see grid below) and a rough reading order from "compare options" through "communicate finished work" to "interactive editors". index.html is the only file that knows about the others — it links to all 20 with thumbnails and category headings.

2
index.html — the gallery

Read this first if you're new. It groups the 20 examples into the six categories described in the README, gives each a short blurb, and links straight to the file. Open it in a browser and you can navigate the whole gallery without touching the filesystem.

3
.claude/skills/html-output/SKILL.md — the skill spec

This is what turns the gallery into a Claude Code skill. SKILL.md has a table that maps "what the user wants" (a PR review, a status report, a postmortem) to the matching template file, then walks Claude through the workflow: pick a template, copy the <style> block verbatim, mirror the section structure, swap placeholder content for real content, output a single self-contained .html.

The skill auto-loads when Claude Code is run from inside this repo. Copy the folder to ~/.claude/skills/ to use it from anywhere — see step 5.

show frontmatter 
--- .claude/skills/html-output/SKILL.md ---
name: html-output
description: Produces a polished, self-contained .html file
  styled in a warm ivory/clay/oat palette for documents like
  PR reviews, code walkthroughs, status reports, incident
  reports, design system pages, slide decks, implementation
  plans, research explainers, flowcharts, and small interactive
  editor UIs.
4
.claude/skills/html-output/templates/ — duplicated on purpose

The 20 files under templates/ are copies of the top-level numbered files. The duplication keeps the two use cases independent: the top-level files are the human-facing gallery (a viewer expects them at the URLs README.md documents), and the templates/ copies are what the skill reads when Claude builds a new artifact. Changing one and forgetting the other will drift the gallery from the skill — keep them in sync.

5
.claude/skills/html-output/references/design-tokens.md

The single source of truth for the palette (--ivory, --slate, --clay, --oat, --olive…) and the serif/sans/mono trio. Every template inlines these tokens at the top of its <style> block, which is why the gallery feels visually consistent despite each file being independent.

  • .claude/skills/html-output/SKILL.md Skill spec — template-picker table and workflow.

  • README.md Project overview and install snippet.

Gotchas

  • Top-level files and templates/ copies are duplicated by design — if you edit one, mirror the change to keep the gallery and skill in sync.
  • No build step and no shared stylesheet: each .html inlines its own tokens. Refactoring "common" CSS out would break the self-contained promise.
  • The skill auto-loads only inside this repo. For global use, copy .claude/skills/html-output to ~/.claude/skills/.
